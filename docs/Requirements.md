Smart Parking Lot System - Requirements

Objective

The goal of this project is to design a backend system for a smart parking lot using Java and Object-Oriented Programming. The system should make it easy to manage vehicle entry and exit, assign parking spots automatically, and calculate parking charges based on how long a vehicle is parked.

Functional Requirements

The system should be able to:

- Register a vehicle when it enters the parking lot.
- Assign an available parking spot based on the type of vehicle.
- Record the entry time of every vehicle.
- Record the exit time when a vehicle leaves.
- Calculate the parking fee using the parking duration.
- Mark the parking spot as available after the vehicle exits.
- Display the available parking spots.
- Support multiple parking floors.
- Handle different vehicle types such as Bike, Car, and Bus.

Non-Functional Requirements

While building this project, the following design principles should be followed:

- Use Object-Oriented Programming concepts.
- Follow SOLID principles wherever applicable.
- Keep the code simple, modular, and easy to understand.
- Separate the responsibilities of different classes.
- Make the system easy to extend with new features in the future.

Assumptions

- One parking spot can hold only one vehicle at a time.
- Each vehicle is assigned only one parking spot.
- Parking charges are calculated based on the total parking duration.
- Parking spots are assigned automatically depending on availability and vehicle type.
